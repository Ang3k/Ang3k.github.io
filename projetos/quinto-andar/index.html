Visão geral

Objetivo e ferramentas

Neste projeto, objetivei desenvolver um conjunto de dados único, para trabalhar com algo mais original e treinar minhas capacidades com APIs e web scraping.

Fiz isso extraindo informações imobiliárias da empresa QuintoAndar em seu site e, posteriormente, realizando análises nos dados coletados.

Utilizei Python e as bibliotecas Pandas, Selenium e BeautifulSoup para o data mining e a criação dos datasets, além de MySQL para as análises e Power BI para a visualização dos insights.

Parte 1

Web scraping e limpeza em Python

1. Contexto de negócio

Roberto, um corretor de imóveis, decide se mudar para o Rio de Janeiro em busca de melhores oportunidades de trabalho. Porém, percebe que as características, valores e itens dos imóveis são diferentes daqueles com os quais está acostumado, reduzindo a efetividade de seu serviço.

Ele encontra uma possível solução ao decidir realizar uma análise de dados no site do QuintoAndar, uma das principais plataformas de venda e aluguel de imóveis, que reúne informações detalhadas sobre valores, interiores e outros atributos.

2. Extração de dados

Primeiramente, busquei uma API para realizar a coleta. Entretanto, a maioria dos sites semelhantes ao QuintoAndar não possuía uma API ou limitava demais as consultas gratuitas. Por isso, optei por realizar web scraping com Python, usando BeautifulSoup4 e Selenium.

Mesmo que inicialmente o projeto tenha sido pensado para utilizar apenas BeautifulSoup, fez-se necessário usar Selenium devido à dinamicidade do site e ao conteúdo carregado com JavaScript.

3. Web scraping em Python

O processo para extrair o conteúdo era simples, mas complexo em execução. Existiam links escondidos nas imagens que redirecionavam para páginas com os dados completos dos imóveis.

Programei o Selenium para atravessar o site e clicar em “Ver mais” por uma quantidade opcional de vezes. Depois, o HTML completo era entregue ao BeautifulSoup. Uma função chamada extrair_links identificava os links escondidos com base em um padrão encontrado durante a exploração.

Esses links continham preço, quantidade de quartos, IPTU e várias outras informações. Criei cerca de onze funções para extrair e limpar os conteúdos. Por exemplo, remover “R$” dos preços e transformar a identificação do imóvel em uma chave numérica.

As informações foram agrupadas em quatro datasets de categorias semelhantes, posteriormente concatenados com os demais arquivos de estados do Sudeste.

Parte 2

Analisando com MySQL e Power BI

A seguir estão as queries mais interessantes do projeto, acompanhadas de contexto e visualizações criadas para tornar os resultados mais claros.

1. Média de preço por tipo de imóvel

A query mostra a variação de preço por tipo de imóvel, organizada em ordem decrescente e agrupada por capital. Belo Horizonte lidera nos preços de casas e apartamentos. Studios são a opção mais barata; apenas em São Paulo apresentam custo semelhante ao de apartamentos.

2. Quantidade de imóveis por tipo

Rio de Janeiro e Belo Horizonte possuem 95,8% de seus imóveis classificados como casas ou apartamentos. Em São Paulo, 8% dos imóveis à venda são studios, fazendo da cidade a líder nessa categoria.

3. Preço e mobília

A consulta demonstra como a presença de mobília afeta tipos de imóveis de maneiras diferentes. Casas sem mobília custam 2,26% a menos; apartamentos, 7,44%; e studios, 17,85%.

4. Preço e data de publicação

Embora o gráfico seja crescente, isso não implica aumento dos preços ao longo do tempo. Indica que anúncios antigos tendem a receber reduções: em média, os imóveis custam cerca de 40,8% menos após um ano de publicação.

5. Principais logradouros por capital

A query mostra os dez logradouros com maior quantidade de imóveis em diferentes capitais, ajudando a evidenciar pontos de oportunidade.

6. Padrão de publicação ao longo da semana

Existe aparentemente um padrão: nos finais de semana, as pessoas costumam publicar mais imóveis. Quarta e quinta-feira são os dias de menor movimento.

7. Custo por metro quadrado

São Paulo é a capital com maior custo por metro quadrado, com destaque para Jardins. Rio de Janeiro e Belo Horizonte disputam a segunda posição, com bairros como Humaitá e Glória entre os mais caros.

8. Itens mais e menos comuns

Para casas e apartamentos, box, área de serviço e armários na cozinha aparecem como itens quase indispensáveis. Em studios, área de serviço é menos comum. Salão de jogos, academia e sauna estão entre os itens mais raros.

Parte 3

Dashboard e conclusão

Para tornar a visualização mais intuitiva, construí um dashboard em que o usuário pode selecionar as capitais que deseja comparar e acessar informações como média de preço por tipo ou flutuação de preços com o tempo de publicação.

Otimização do código e escolha da página

Meu código de web scraping não era muito otimizado, pois as operações demoravam bastante. A página do QuintoAndar, embora excelente em conteúdo, carregava todos os imóveis no mesmo lugar, tornando o processo lento e consumindo muita memória RAM.

Planejamento das informações

Pensei que seria capaz de extrair mais insights com os dados obtidos. No projeto anterior, consegui gerar quinze análises de qualidade satisfatória; neste, foram oito. Isso se deve tanto à minha experiência inicial quanto à quantidade e variedade dos dados coletados.

Melhoria nas técnicas

Nos primeiros projetos, utilizei principalmente tecnologias do campo da análise de dados, como SQL e Power BI. A partir daqui, passei a buscar análises mais profundas com Machine Learning em Python, tornando os estudos mais detalhados e estimulantes.

Sendo minha primeira experiência prática com web scraping, localizei as principais características de cada anúncio, consegui extraí-las e analisá-las com bom grau de clareza e profundidade.
